Project Sustainable Business Network 7: Mapping Trade Show Vendors, DS 4200 S20

Samantha Hoch, Ankita Uppugunduri, Kees van Hemmen

Service-Learning Course Project as part of DS 4200: Information Presentation and Visualization, taught by Prof. Dylan Cashman, Data Visualization @ Khoury, Northeastern University.

Motivation

The Sustainable Business Network of Massachusetts hosts an annual trade show to help connect local vendors with restaurants and markets. This visualization will help buyers better understand the types of vendors that typically attend the show and where the vendors come from. By making this data more accessible, buyers will be more inclined to attend the show.

Data Analysis

Summary of data, data types, and data preprocessing.

Expectation: Data sources match the problem statement and are appropriate. These descriptions should be very explicit so someone could read your page and properly reproduce your results.

The data we are using for our interactive visualization is the Local Food Trade Show Vendor Application Specialty Crops dataset. The dataset has 20 columns, each of which is either a numerical value (ex.: zip code), text (ex.: products sold), or some combination of both (columns wherein a numerical value is specified, no is specified, or n/a). In order to make this data more usable we first indexed the data points (there was no primary key with which we could individually refer to rows by). We also standardized inputs in every column for effective grouping. This meant standardizing the N/A value (it was stored as na, N/A, and n/a in various places) and making any percentage value appear as a whole value, as opposed to a decimal, in the columns needed for our visualization. These columns were as follows: “Approximately what percentage of your products do you currently sell wholesale?”, “Do you have a goal for what percentage of your products you would like to sell wholesale?”, and “How long has your company been in business?”. Editing the zipcode columns was ultimately not necessary as the data was properly standardized. After the data cleaning was complete we converted the excel files into JSONs in the event that we would need them in that format.
